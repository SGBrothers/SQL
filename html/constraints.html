Defined Constraint

Constraints in SQL means we are applying certain conditions or restrictions on the database. This further means that before inserting data into the database, we are checking for some conditions. If the condition we have applied to the database holds true for the data which is to be inserted, then only the data will be inserted into the database tables.

Constraints can be specified when the table is created with the CREATE TABLE statement, or after the table is created with the ALTER TABLE statement.

Constraints in SQL can be categorized into two types:

1. Column Level Constraint : A Column Level Constraint can only be defined when we are defining the column in the table.

2. Table Level Constraint : A Table Level Constraint can be defined when we are creating the table or modified the structured of the table.

There are different types of constraint

1. Primary Key constraint
2. Not null constraint
3. Unique constraint
4. Check constraint
5. Default constraint
6. Foreign key constraint

Primary key Constraint

The PRIMARY KEY constraint uniquely identifies each record in a table.

Primary keys must contain UNIQUE values, and cannot contain NULL values.

A table can have only ONE primary key; and in the table, this primary key can consist of single or multiple columns (fields).

A primary key constraint is the column level as well as Table level constraint.

Syntax of Primary Key Constraint

CREATE TABLE TableName (
ColumnName1 datatype,
ColumnName2 datatype,
….,
ColumnNameN datatype,
constraint constraint_name primary key(ColumnName)
);

Example of Primary Key Constraint

Syntax of Drop primary Key Constraint

Mysql> ALTER TABLE table_name DROP Constraint constraint_name ;

Example of Drop primary Key Constraint

Not Null constraint

Not Null is a column level constraint.

when not null is defined then we cannot give null values in that column.

This enforces a field to always contain a value, which means that you cannot insert a new record, or update a record without adding a value to this field.

Syntax Of Not Null constraint

Mysql> CREATE TABLE TableName (
ColumnName1 datatype NOT NULL,
ColumnName2 datatype,
….,
ColumnNameN datatype
);

Example of Not Null constraint

Unique Constraint

Duplicate values are not allowed in the columns to which the UNIQUE constraint is applied.

The column with the unique constraint will always contain a unique value.

Using the UNIQUE constraint, you can also modify the already created tables.

This constraint can be applied to one or more than one column of a table, which means more than one unique constraint can exist on a single table.

Syntax Of Unique Constraint

Mysql> CREATE TABLE TableName (
ColumnName1 datatype UNIQUE,
ColumnName2 datatype,
….,
ColumnNameN datatype
);

Example of Unique Constraint

Check Constraint

The CHECK constraint is used to limit the value range that can be placed in a column.

If you define a CHECK constraint on a table it can limit the values in certain columns based on values in other columns in the row.

Whenever a check constraint is applied to the table's column, and the user wants to insert the value in it, then the value will first be checked for certain conditions before inserting the value into that column.

Syntax Of Check Constraint

Mysql> CREATE TABLE TableName (
ColumnName1 datatype CHECK (ColumnName1 Condition),
ColumnName2 datatype,
….,
ColumnNameN datatype
);

Example of check constraint

Default Constraint

The DEFAULT constraint is used to set a default value for a column.

The default value will be added to all new records, if no other value is specified.

Whenever a default constraint is applied to the table's column, and the user has not specified the value to be inserted in it, then the default value which was specified while applying the default constraint will be inserted into that particular column.

Syntax Of Default Constraint

Mysql> CREATE TABLE TableName (
ColumnName1 datatype DEFAULT Value,
ColumnName2 datatype,
….,
ColumnNameN datatype
);

Example of Default Constraint

Foreign Key Constraint

Foreign key constraint is also Known as Referential Integrity Constraint.

It maintains the integrity data by checking the refrence of the data from the primary key table.

The table on which we have Primary key is Known as Parent table/ Master table / Primary table.

The table on which on have Foreign key is also Known as Child table/ Slave table/ Secondary table.

The column of the master table and the slave table datatype should be same (column name can be different

The table can have Multiple foreign key.

If the master table we are having the two column on which primary key is apply then is a slave table then should be two which are refrence to the master table column.

Syntax Of Foreign Key Constraint

Mysql>CREATE TABLE tablename(
ColumnName1 Datatype ,
ColumnNameN Datatype,
FOREIGN KEY( ColumnName ) REFERENCES PARENT_TABLE_NAME(Primary_Key_ColumnName)
);

Mysql> ALTER TABLE table_name ADD CONSTRAINT constraint_name FOREIGN KEY(columName)
REFERENCES PARENT_TABLE_NAME(Primary_Key_ColumnName) ;

Example of Foreign Key Constraint

Foreign Key Constraint Parameter

There are Four Types of Foreign Key Constraint Parameter :

1. On delete set null
2. On delete cascade
3. On update set null
4. On update cascade

1. ON DELETE SET NULL :- A foreign key with "set null on delete" means that if a record in the parent table is deleted, then the corresponding records in the child table will have the foreign key fields set to NULL.

Syntax Of On delete set null

Mysql> CREATE TABLE table_name(
Column1 datatype,
Column2 datatype,
FOREIGN KEY(ColumnName)
REFERENCES PARENT_TABLE(ColumnName) ON DELETE SET NULL
);


2. ON UPDATE SET NULL :I know that when setting cascade actions for a table, we may use ON UPDATE SET NULL to, well, set the foreign key field to null when the referenced row is updated.

Syntax Of On update set null

Mysql> CREATE TABLE table_name(
Column1 datatype,
Column2 datatype,
FOREIGN KEY(ColumnName)
REFERENCES PARENT_TABLE(ColumnName) ON UPDATE SET NULL
);

Example of ON DELETE SET NULL and ON UPDATE SET NULL

3. ON DELETE CASCADE :DELETE CASCADE Indicates that when the row in the parent table is deleted, the dependent rows in the child table will also be deleted

Syntax Of delete cascade

Mysql> CREATE TABLE table_name(
Column1 datatype,
Column2 datatype,
FOREIGN KEY(ColumnName)
REFERENCES PARENT_TABLE(ColumnName) ON DELETE CASCADE );

4. ON UPDATE CASCADE : ON UPDATE CASCADE clause in MySQL is used to update the matching records from the child table automatically when we update the rows in the parent table.

Syntax Of On update cascade

Mysql> CREATE TABLE table_name(
Column1 datatype,
Column2 datatype,
FOREIGN KEY(ColumnName)
REFERENCES PARENT_TABLE(ColumnName) ON UPDATE CASCADE
);


Example of ON DELETE CASCADE and ON UPDATE CASCADE
